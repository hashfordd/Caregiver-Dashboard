# F9 — Outdoor GPS Map View

## Goal

When the patient is outdoors, the dashboard switches from the indoor floor plan to a Mapbox-backed map showing the latest GPS pin, a breadcrumb trail of the last 30 minutes of outdoor positions, and an editable geofence polygon. The view is a pure consumer of `position_estimates` rows where `mode = 'outdoor'`; the indoor↔outdoor switch logic itself lives in F8 / POS-08. F9's responsibility ends at letting the caregiver draw and persist a geofence — the actual rule-firing on geofence breach is Phase 4 / [F11](./F11.md).

## Acceptance criteria

- GPS pin updates within 5 s of a new outdoor `position_estimates` row arriving (TST-06).
- Breadcrumb persists the last 30 minutes of `mode = 'outdoor'` estimates as a fading polyline.
- Mode-switch UI: when `position_estimates.mode === 'outdoor'`, the patient detail tab switches its position view from the floor plan (F8) to this map. When `mode === 'indoor'`, the map view is collapsed/hidden; the floor plan resumes. The switch itself is hysteretic — driven by F8's [POS-08 logic](./F8.md#mode-switch-hysteresis-pos-08), not re-implemented here.
- Geofence polygon: caregiver can draw, edit, and delete a polygon overlaid on the map; the polygon persists to `alert_rules.params` for a `type = 'zone'` rule. The actual rule firing on breach is Phase 4 / [F11](./F11.md) — F9 owns the draw/persist surface only.

## Dependencies

- **Hard**:

  - F8 must be writing `position_estimates` rows with `mode`, `lat`, `lng`, `confidence` populated. F9 is a consumer.
  - Mapbox token in dev env: `VITE_MAPBOX_TOKEN`. Document in `apps/web/.env.example`.
  - Mock telemetry generator with GPS fixes in its signals payloads (POS-10 replay mode includes outdoor sequences).

- **Cross-cutting**:

  - [CROSS_CUTTING §7 — Realtime patterns](../CROSS_CUTTING.md#7-realtime-patterns): consume `position_estimates` via the existing `usePatientStream` hook. No new realtime hook.
  - [CROSS_CUTTING §16 — Performance budgets](../CROSS_CUTTING.md#16-performance-budgets): Mapbox adds ~100 KB gzipped. Lazy-load the map module per route; the bundle budget allows it but only if not eagerly imported.
  - [CROSS_CUTTING §4 — Loading / empty / error / stale states](../CROSS_CUTTING.md#4-loading--empty--error--stale-states): map handles loading (Mapbox style fetch), empty (no recent outdoor estimates → "Patient is indoors. Map will activate when outdoor"), error (token missing or fetch failed), and stale (last GPS fix > 30 s old → yellow pip on the pin).
  - [CROSS_CUTTING §1 — RLS write policies](../CROSS_CUTTING.md#1-rls-write-policies): geofence polygon writes go to `alert_rules` — caregiver-write policy already specified there.

- **What this unblocks**: [F11](./F11.md) consumes the `alert_rules.params.geofence` polygon shape locked here for the zone rule type. F11 doesn't exist yet — link is forward-referenced; the next batch creates it.

## Contracts in packages/shared

The geofence polygon payload shape, locked here. F11 reads from it.

```ts
// packages/shared/src/rules/geofence.ts (new file; rules/ dir created here, expanded in F11)

export type GeofencePolygon = {
  type: 'polygon';
  // Closed polygon: first and last points must be equal.
  // Coordinates are [lng, lat] pairs (GeoJSON convention) for outdoor zones.
  coordinates: [number, number][];
};

export type GeofenceParams = {
  geofence: GeofencePolygon;
  // 'enter' fires when the patient crosses INTO the polygon;
  // 'exit' fires when they cross OUT.
  mode: 'enter' | 'exit';
  // Per-rule cooldown override; falls back to severity default per CROSS_CUTTING §3.
  cooldown_seconds?: number;
};

export type ZoneAlertRule = {
  id: string;
  patient_id: string;
  type: 'zone';
  params: GeofenceParams;
  severity: 'info' | 'warn' | 'critical';
  enabled: boolean;
};
```

Re-export from `packages/shared/src/index.ts`. F9 writes this shape; F11 reads it.

**Note on coordinate convention**: GeoJSON is `[lng, lat]`, opposite of how most humans say it. Mapbox expects GeoJSON. Lock `[lng, lat]` here; F11 must use the same order. Add a brief comment block in `geofence.ts` to that effect.

**Indoor zones (canvas polygons) are deferred to F11**. F9 only handles outdoor / lat-lng geofences. Indoor zone rules (forbidden room, bed area) use canvas coordinates and a different polygon type — F11 will extend this with a discriminated union.

## Risks

- **Mapbox cost at scale**. Free tier (50 K monthly map loads) is fine for prototype demo. Document this in BACKLOG as a V2 concern. Mitigation: lazy-load the map; don't initialise it on the roster page.
- **GPS drift indoors near doorways**. Without F8's hysteresis (POS-08), the view would flap between map and floor plan whenever the patient stood near a window with an intermittent GPS fix. F9 trusts F8 to suppress this; F9's only defence is to render whatever `mode` says. If F8's hysteresis is broken, F9 will visibly thrash — that's the symptom that catches the bug.
- **Mapbox style fetch network failure**. The map silently shows a grey background. Mitigation: explicit error handling on the Mapbox `error` event; fall back to the placeholder per [CROSS_CUTTING §4](../CROSS_CUTTING.md#4-loading--empty--error--stale-states).
- **Breadcrumb cache coherence**. Two strategies for the 30 min trail:
  - **(a)** React Query fetches the window on mount + on each new outdoor estimate. Simple but does a fetch per arriving point.
  - **(b)** Initial fetch on mount; append to a Zustand slice on each realtime arrival; trim the slice to 30 min. No re-fetch storms.
  - **Recommendation**: (b). Consistent with [CROSS_CUTTING §7](../CROSS_CUTTING.md#7-realtime-patterns) — live data via Zustand, not React Query.
- **Polygon coordinate convention drift**. F9 stores `[lng, lat]`; F11 must read `[lng, lat]`. If F11 confuses the order it'll fire alerts in the wrong place. Mitigation: type the contract sharply (above), add a Vitest in F11 that round-trips a known polygon through the rule evaluator.
- **`@mapbox/mapbox-gl-draw` peer-dep mismatch**. The draw library tracks Mapbox GL JS major versions; verify the pinned versions are compatible at install time. If not, fall back to a custom polygon UI (click vertices, double-click to close, drag handles to edit).
